# Plan: PRJ-004 – Docker Compose einrichten

**Status: Implementiert**

## Was wurde erstellt

| Datei | Beschreibung |
|-------|--------------|
| `docker-compose.yml` (Root) | Alle 4 Services: MSSQL 2022, Redis 7, Backend, Frontend |
| `backend/Dockerfile` | Multi-stage: Gradle Build → JRE Runtime |
| `backend/.dockerignore` | Schließt build/, .gradle/, .idea/ aus |
| `frontend/Dockerfile` | Multi-stage: Node Build → nginx |
| `frontend/nginx.conf` | Hört auf Port 80, Angular-Router-Fallback |
| `frontend/.dockerignore` | Schließt node_modules/, dist/, .angular/ aus |

## Entscheidungen

- **MSSQL statt PostgreSQL**: Ticket-Widerspruch (Services: "PostgreSQL", Volumes: "MSSQL") → MSSQL gemäß PRJ-003
- **Keine .env.example**: Dev-Werte stehen direkt in docker-compose.yml; Production-Secrets kommen aus CI/CD-Pipeline
- **nginx port 80 intern, 4200 extern**: `docker compose up` → `localhost:4200` öffnet die App

## Verifikation

```bash
docker compose build
docker compose up -d
docker compose ps
curl http://localhost:8080/actuator/health
# Browser: http://localhost:4200
```
