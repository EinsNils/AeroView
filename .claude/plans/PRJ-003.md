# Plan: PRJ-003 – Spring Boot Backend initialisieren

## Context
Issue #3 verlangt das vollständige Setup des Spring Boot Backends. Aktuell enthält das Backend nur den minimalen `spring-boot-starter` ohne Web, JPA, Security etc. Alle Starter, die Ordnerstruktur, `application.yml` und Flyway müssen von Grund auf ergänzt werden.

**Status: Implementiert**

---

## Implementierte Änderungen

### build.gradle.kts
- Web, WebSocket, JPA, MSSQL JDBC, Flyway (SQL Server), Security, Redis, Actuator
- MapStruct 1.6.3 + Lombok (Lombok vor MapStruct in annotationProcessor)

### application.yaml
- MSSQL JDBC-URL mit `createDatabaseIfNotExist=true`
- Redis-Konfiguration
- Flyway auf `classpath:db/migration`
- Actuator health endpoint exposed

### docker-compose.yml (`backend/`)
- MSSQL 2022 + Redis 7 Alpine
- `docker compose up -d` startet beide Services

### Ordnerstruktur
- `auth/`, `flights/`, `airports/`, `user/`, `ws/`, `integration/` unter `dev.nz3x.backend`

### Flyway
- `src/main/resources/db/migration/V1__init.sql` als leere Initials-Migration
