## Workflow-Struktur

Ein `ci.yml` mit 3 Jobs:
1. **frontend** – `npm run build` → `npm test --watch=false` → `npm run lint`
2. **backend** – `./gradlew test` (mit Gradle-Cache via `gradle/actions/setup-gradle`)
3. **docker-publish** – baut Backend + Frontend Image, pusht zu ghcr.io
   - Läuft nur bei Push auf `main`
   - `needs: [frontend, backend]` — nur wenn Tests grün
   - Images: `ghcr.io/einsnils/aeroview-backend` + `ghcr.io/einsnils/aeroview-frontend`
   - Tags: `latest` + `sha-<commit-sha>`
   - GHA-Cache für schnellere Rebuilds
